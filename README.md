ShellSage intercepts Bash-style tool calls made by your AI coding agent (Claude Code, VSCode/GitHub Copilot, Kiro, Cursor, Cline ‚Ä¶) and silently rewrites bash syntax into correct PowerShell/CMD before the shell sees it. It works immediately with a local rule engine; optional Qdrant vector memory can learn from successful sessions.

**No API key. No cloud. Runs entirely on your machine.**

---

## What it does

| Without ShellSage | With ShellSage |
|---|---|
| Agent writes `ls -la` ‚Üí PowerShell fails ‚Üí retry loop ‚Üí 45k wasted tokens | Agent writes `ls -la` ‚Üí silently becomes `Get-ChildItem -Force` ‚Üí works ‚úď |
| 3 bash failures per session = ~135k wasted tokens | 0 failures ¬∑ 0 wasted tokens |
| Error traces pollute all future turns | Errors never reach the LLM context |

**How it translates:**

1. **Rule-based translation** ‚ÄĒ 100+ regex patterns covering common bash constructs. Zero Qdrant dependency and fast on first run.
2. **Optional Qdrant hybrid search** ‚ÄĒ semantic embedding + BM25 lexical search fused via Reciprocal Rank Fusion (RRF). Loads a bounded curated seed set by default and can learn from past sessions.
3. **Passthrough** ‚ÄĒ if no translation is needed (bash on Linux/macOS, or git/docker commands that work everywhere), the command passes through unchanged.

---

## Quickstart

```bash
# 1. Install the lightweight package
pip install "shellsage[mcp]"

# 2. Register with Claude Code
claude mcp add shellsage -- shellsage mcp

# 3. Install hooks in your project
shellsage hooks install
```

That's it. ShellSage now intercepts every Bash call.

Optional vector memory:

```bash
pip install "shellsage[vector]"
docker compose up -d
shellsage init          # loads a limited curated seed set with a progress bar
shellsage init --all    # loads the complete seed corpus
```

## Setup Guides

### Claude Code (recommended ‚ÄĒ hooks + MCP)

Claude Code supports **hooks** (silently rewrite before execution) and **MCP** (tools the model can call).

#### Step 1 ‚ÄĒ Register the MCP server

```bash
claude mcp add shellsage -- shellsage mcp
```

#### Step 2 ‚ÄĒ Install project hooks

Run inside your project directory:

```bash
shellsage hooks install
```

This creates `.claude/hooks/pre_tool_use.py` and `.claude/hooks/post_tool_use.py`.

Add this to `.claude/settings.json`:

```json
{
  "hooks": {
    "PreToolUse":  [{"matcher": "Bash", "hooks": [{"type": "command", "command": "python .claude/hooks/pre_tool_use.py"}]}],
    "PostToolUse": [{"matcher": "Bash", "hooks": [{"type": "command", "command": "python .claude/hooks/post_tool_use.py"}]}]
  }
}
```

**What each hook does:**
- `pre_tool_use.py` ‚ÄĒ translates the command before execution; writes original‚Üítranslated to a temp cache
- `post_tool_use.py` ‚ÄĒ reads the cache and records success/failure to Qdrant when vector memory is installed

### VSCode / GitHub Copilot / Kiro / Claude Code MCP

Add to your MCP settings (e.g. `cline_mcp_settings.json` or cursor MCP panel):

```json
{
  "mcpServers": {
    "shellsage": {
      "command": "shellsage",
      "args": ["mcp"]
    }
  }
}
```

### Environment variable (optional)

Point ShellSage at a remote or differently-ported Qdrant:

```bash
export SHELLSAGE_QDRANT_URL=http://my-qdrant-host:6333
```

## Configuration

All settings can be overridden via environment variables:

| Variable | Default | Description |
|---|---|---|
| `SHELLSAGE_QDRANT_URL` | `http://localhost:6333` | Qdrant server URL |
| `SHELLSAGE_SCORE_THRESHOLD` | `0.82` | Minimum cosine similarity to accept a Qdrant hit |
| `SHELLSAGE_EMBED_MODEL` | `all-MiniLM-L6-v2` | Sentence-transformer model (384-dim, ~22 MB) |
| `SHELLSAGE_SEED_LIMIT` | `75` | Number of seed examples loaded by default |
| `SHELLSAGE_SEED_CONFIDENCE` | `0.95` | Confidence assigned to seed translations |
| `SHELLSAGE_OUTCOME_CONFIDENCE` | `0.99` | Confidence assigned when a command succeeds in practice |

Example:

```bash
export SHELLSAGE_QDRANT_URL=http://my-server:6333
export SHELLSAGE_SCORE_THRESHOLD=0.85
shellsage init
```

## CLI Reference

```
shellsage init                       # Create collections + load a limited seed set
shellsage init --all                 # Load the complete seed corpus
shellsage translate "ls -la"         # Translate a single command (--json-out for JSON)
shellsage stats                      # Show Qdrant collection counts
shellsage replay                     # Show recent failure patterns
shellsage mcp                        # Start MCP server
shellsage hooks install              # Write pre/post hook scripts to .claude/hooks/
shellsage --version                  # Print version
```

| Module | Role |
|---|---|

| `rules.py` | 100+ regex patterns (cold-start, no Qdrant needed) |
| `seed.py` | Curated bash‚ÜíPS translations; `init` loads a bounded set by default |
| `embedder.py` | Lazy `all-MiniLM-L6-v2` (384-dim, 22 MB, CPU) |
| `store.py` | Qdrant: 3 collections, hybrid search (BM25 + cosine + RRF) |
| `translator.py` | 3-tier resolution: rules ‚Üí optional Qdrant ‚Üí passthrough |
| `server.py` | FastMCP server (4 tools) |
| `cli.py` | Click CLI (6 commands) |
