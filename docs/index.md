## What is ShellSage?

ShellSage intercepts Bash-style tool calls made by your AI coding agent ‚ÄĒ Claude Code, Cursor, Windsurf, Kiro, Cline ‚ÄĒ and **silently rewrites bash syntax into correct PowerShell/CMD** before the shell sees it.

It works immediately with a local rule engine and a SQLite memory that learns from your sessions. **No API key. No cloud. No Docker. Runs entirely on your machine.**

---

## Why it matters

| Without ShellSage | With ShellSage |
|---|---|
| Agent writes `ls -la` ‚Üí PowerShell fails ‚Üí retry loop ‚Üí 45k wasted tokens | Agent writes `ls -la` ‚Üí silently becomes `Get-ChildItem -Force` ‚Üí works ‚úď |
| 3 bash failures per session ‚Čą 135k wasted tokens | 0 failures ¬∑ 0 wasted tokens |
| Error traces pollute all future turns | Errors never reach the LLM context |

## How it translates

```mermaid
flowchart LR
    A["AI Agent<br/>(bash command)"] --> B["Rule Engine<br/>100+ regex patterns"]
    B -->|hit| D["‚úÖ PowerShell"]
    B -->|miss| C["SQLite Memory<br/>400+ curated seeds +<br/>session-learned"]
    C -->|hit| D
    C -->|miss| E["Passthrough<br/>(already native)"]
    E --> D
```

1. **Rule-based translation** ‚ÄĒ 100+ regex patterns covering common bash constructs. Instant, zero DB dependency.
2. **SQLite memory** ‚ÄĒ BM25-style lookup over 400+ curated seed translations plus anything learned from your own sessions. Stored locally in `~/.shellsage/memory.db`.
3. **Passthrough** ‚ÄĒ if no translation is needed (native PowerShell, git, docker), the command passes through unchanged.

## Quick install

```bash
pip install "shellsage[mcp]"
shellsage setup
```

The setup wizard detects your IDE, seeds the local database, and registers the MCP server in under a minute.
